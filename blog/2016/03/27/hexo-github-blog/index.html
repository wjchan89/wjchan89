Hexo + github 搭建属于自己的博客

发表于 2016-03-27   |   分类于 博客   |     |   阅读次数

一直想维护一个博客，纪录下些什么，主要是做笔记吧。今天找了些时间搭了起来。总体还是很顺利的。以下是在 mac 环境下的操作。

配置环境

安装 Node.js

到 Node.js 官网下载相应平台的最新版本，一路安装即可。

安装 Git

git 主要用于提交静态代码到 github ，mac 装了 xcode 就会自带 git 的。
到 github 官网申请 git 帐号，主要用作博客的远程仓库、服务器。

hexo 插件

自动生成 sitemap，RSS，部署到 git 等，建议安装。

建站

新建完成后，指定文件夹的目录如下：

_config.yml
网站的配置信息，可以在此配置大部分的参数。

package.json
应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，可以自由移除。

scaffolds
模版文件夹。当新建文章时，Hexo 会根据 scaffold 来建立文件。

source
资源文件夹是存放用户资源的地方。除 posts 文件夹之外，开头命名为 (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes
主题文件夹。Hexo 会根据主题来生成静态页面。

启动本地服务器

在博客工程根目录下执行，会生成静态文件 public 目录下。也可用完整命令 hexo generate

执行后，浏览器输入 http://localhost:4000 ，简单版的博客出现了。完整版命令 hexo server，也可加调试参数 –debug。
