站内搜索

本站用了 swiftype ，next主题也集成了 swiftype 搜索，需要到 swiftype 配置一个搜索引擎， 而后编辑主题配置文件， 修改 swiftype_key 字段，值为你的 swiftype 搜索引擎的 key。 详细的配置请参考：swiftype详细教程

  1. 进入leancloud官网注册一个帐号
  2. 创建一个应用，名字随意
  3. 进入应用，选择存储一栏，点击数据旁边的设置图案，选择创建Class
  4. 在Class name一栏输入 Counter 后创建
  5. 点击菜单栏中的设置，在应用 URL 填入自己的博客地址后保存，点击左侧应用 Key 一栏
  6. 将APP Id和APP Key复制，填入博客中主题配置文件 _config.yml 中：
  1. 重新部署发布博客，记录文章访问量的唯一标识符是文章的发布日期以及文章的标题，因此请确保这两个数值组合的唯一性，如果更改了这两个数值，会造成文章阅读数值的清零重计
  2. 为了确保该应用只在自己的博客上使用，在安全中心中添加 web 安全域名，填入自己博客域名，这样就只有自己的博客可以访问后台数据了

Hexo + github 搭建属于自己的博客

发表于 2016-03-27   |   分类于 博客   |  

一直想维护一个博客，纪录下些什么，主要是做笔记吧。今天找了些时间搭了起来。总体还是很顺利的。以下是在 mac 环境下的操作。

配置环境

安装 Node.js

到 Node.js 官网下载相应平台的最新版本，一路安装即可。

安装 Git

git 主要用于提交静态代码到 github ，mac 装了 xcode 就会自带 git 的。
到 github 官网申请 git 帐号，主要用作博客的远程仓库、服务器。

hexo 插件

自动生成 sitemap，RSS，部署到 git 等，建议安装。

建站

新建完成后，指定文件夹的目录如下：

_config.yml
网站的配置信息，可以在此配置大部分的参数。

package.json
应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，可以自由移除。

scaffolds
模版文件夹。当新建文章时，Hexo 会根据 scaffold 来建立文件。

source
资源文件夹是存放用户资源的地方。除 posts 文件夹之外，开头命名为 (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes
主题文件夹。Hexo 会根据主题来生成静态页面。

启动本地服务器

在博客工程根目录下执行，会生成静态文件 public 目录下。也可用完整命令 hexo generate

执行后，浏览器输入 http://localhost:4000 ，简单版的博客出现了。完整版命令 hexo server，也可加调试参数 –debug。
